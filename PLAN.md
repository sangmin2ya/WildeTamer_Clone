# 슈퍼메이커 과제 — 4일 개발 계획

## 프로젝트 개요

- **레퍼런스 게임**: Wild Tamer
- **엔진**: Unity (2D, Rigidbody2D Kinematic 기반)
- **시점**: 쿼터뷰 (Isometric)
- **입력**: PC WASD 기준 (New Input System)
- **핵심 루프**: 탐험 → 전투 → 테이밍 → 부대 확장

---

## 아키텍처 원칙

- **오브젝트 풀링**: 몬스터, 이펙트, UI 팝업, 발사체 등 모든 동적 오브젝트는 Day 1에 구축한 풀링 매니저를 통해서만 생성/반환한다
- **NavMesh 미사용**: 모든 이동은 순수 코드로 구현한다. `Rigidbody2D(Kinematic)` + `MovePosition` 방식을 사용하며, 벽/지형 충돌은 Collider2D가 자동 처리한다
- **유닛 간 겹침 방지**: 물리 충돌 대신 Separation Vector를 MovePosition 이동 벡터에 합산하는 방식으로 처리한다. Collider2D는 벽 충돌 전용으로만 사용한다
- **레이어 구성**: `Wall` / `Ally` / `Enemy` 레이어를 분리하고, Ally↔Enemy 간 물리 충돌은 OFF, Wall과의 충돌만 ON으로 설정한다
- **ScriptableObject**: 몬스터/유닛 스탯은 ScriptableObject로 정의하며, 적 상태와 아군 상태 스탯을 분리한다
- **공간 분할**: 유닛 탐색은 SpatialHashGrid를 통해 수행한다 (Physics.OverlapCircle 남용 금지)
- **상태머신**: 모든 AI(아군/적 유닛, 보스)는 명시적 상태머신으로 관리한다
